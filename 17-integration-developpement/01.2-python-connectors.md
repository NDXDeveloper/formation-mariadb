# 17.1.2 Python : mysql-connector, PyMySQL, SQLAlchemy

L'écosystème Python distingue deux niveaux d'outils, qu'il ne faut pas confondre :

- les **pilotes bas niveau**, qui implémentent l'interface standard **DB-API 2.0** (PEP 249) : ils ouvrent une connexion, exécutent du SQL et renvoient des lignes. PyMySQL et mysql-connector-python appartiennent à cette catégorie ;
- **SQLAlchemy**, qui n'est pas un pilote mais une **couche d'abstraction** (boîte à outils SQL + ORM) posée *au-dessus* d'un pilote DB-API.

Cette section présente les pilotes, puis SQLAlchemy sous l'angle de la connexion. L'usage de SQLAlchemy comme ORM (modèles, sessions, requêtes objet) est traité au §17.3.2.

---

## Le socle commun : DB-API 2.0

Tous les pilotes Python pour MariaDB exposent le même schéma d'utilisation, défini par la PEP 249 :

```python
conn = pilote.connect(...)     # ouverture de la connexion
cursor = conn.cursor()         # création d'un curseur
cursor.execute(requete, parametres)
lignes = cursor.fetchall()     # ou fetchone(), ou itération
cursor.close()
conn.close()
```

### Le marqueur de paramètre — point de sécurité critique

La plupart des pilotes Python utilisent `%s` comme marqueur de paramètre (style *format* de la DB-API). **Ce `%s` n'est pas le formatage de chaîne de Python.** Il doit toujours être confié au pilote via le second argument d'`execute()`, jamais interpolé manuellement :

```python
# ✅ CORRECT — le pilote échappe et lie la valeur
cursor.execute(
    "SELECT id, nom FROM produits WHERE nom = %s",
    (nom,),                       # tuple : noter la virgule finale
)

# ❌ DANGER — injection SQL : ne JAMAIS faire cela
cursor.execute("SELECT id, nom FROM produits WHERE nom = '%s'" % nom)
cursor.execute(f"SELECT id, nom FROM produits WHERE nom = '{nom}'")
```

C'est l'erreur la plus fréquente en Python : confondre le marqueur DB-API avec l'opérateur `%` ou une f-string. Le sujet est approfondi en §17.8 (injections) et §17.9 (*prepared statements*).

> ⚠️ **Attention** : le marqueur diffère selon le pilote. PyMySQL et mysql-connector-python utilisent `%s` ; **MariaDB Connector/Python utilise `?`** (voir plus bas). Un code déplacé d'un pilote à l'autre cassera sur ce point.

## PyMySQL

Implémentation **entièrement en Python**, sans aucune dépendance native à compiler : c'est l'option la plus simple à installer (notamment en environnement contraint ou conteneurisé léger).

```python
import os
import pymysql
import pymysql.cursors

conn = pymysql.connect(
    host="localhost",
    port=3306,
    user="app_user",
    password=os.environ["DB_PASSWORD"],
    database="boutique",
    charset="utf8mb4",
    cursorclass=pymysql.cursors.DictCursor,   # lignes sous forme de dict
)

try:
    with conn.cursor() as cursor:
        cursor.execute(
            "SELECT id, nom, prix FROM produits WHERE categorie = %s AND prix < %s",
            (categorie, prix_max),
        )
        for row in cursor.fetchall():
            print(f"{row['nom']} : {row['prix']:.2f} €")
finally:
    conn.close()
```

Le curseur s'utilise comme gestionnaire de contexte (`with conn.cursor() as cursor`), ce qui garantit sa fermeture.

## mysql-connector-python

Connecteur **officiel publié par Oracle** (paquet `mysql-connector-python`, importé sous `mysql.connector`). Conforme à la DB-API, il existe en version pure Python et avec une extension C optionnelle (paramètre `use_pure`).

```python
import os
import mysql.connector

conn = mysql.connector.connect(
    host="localhost",
    port=3306,
    user="app_user",
    password=os.environ["DB_PASSWORD"],
    database="boutique",
    charset="utf8mb4",
)

cursor = conn.cursor(dictionary=True)   # lignes sous forme de dict
cursor.execute(
    "SELECT id, nom, prix FROM produits WHERE categorie = %s AND prix < %s",
    (categorie, prix_max),
)
for row in cursor:
    print(f"{row['nom']} : {row['prix']:.2f} €")

cursor.close()
conn.close()
```

Conçu d'abord pour MySQL, il prend nativement en charge le plugin `caching_sha2_password`, ce qui le rend commode face à un MariaDB 12.3 (qui supporte désormais ce plugin, §10.5.5).

## Les autres pilotes

Deux options méritent d'être connues bien qu'absentes du titre de cette section :

- **mysqlclient** — extension C s'appuyant sur `libmariadb`/`libmysqlclient`, héritière de l'ancien MySQLdb. Très performante, c'est le pilote **recommandé par défaut pour Django**. Marqueur `%s`. Nécessite des outils de compilation et la bibliothèque cliente à l'installation.
- **MariaDB Connector/Python** — connecteur **officiel de MariaDB** (paquet `mariadb`), bâti sur MariaDB Connector/C. Particularité importante : il utilise le marqueur **`?`** (style *qmark*) et non `%s`. C'est aussi l'option qui prend nativement en charge les plugins d'authentification propres à MariaDB, notamment **`ed25519`**.

---

## Autocommit : un piège classique

Par défaut, ces pilotes ouvrent la connexion avec l'**autocommit désactivé**. Une instruction `INSERT`, `UPDATE` ou `DELETE` n'est donc pas persistée tant que `conn.commit()` n'a pas été appelé ; à la fermeture de la connexion, les modifications non validées sont annulées (avec InnoDB).

```python
cursor.execute(
    "INSERT INTO produits (nom, prix, categorie) VALUES (%s, %s, %s)",
    ("Clavier", 49.90, "peripheriques"),
)
conn.commit()        # sans cette ligne, l'insertion est perdue
```

On peut activer l'autocommit (`conn.autocommit = True` ou paramètre à la connexion) pour les opérations isolées, mais il est généralement préférable de maîtriser explicitement les transactions (chap. 6).

## SQLAlchemy : la couche d'abstraction

SQLAlchemy se distingue radicalement des pilotes ci-dessus : c'est une **boîte à outils SQL et un ORM**. Au niveau connexion, on retient deux apports majeurs : une **URL de connexion** unifiée et un **pool de connexions intégré**.

### Engine et URL de connexion

Le point d'entrée est l'`Engine`, construit à partir d'une URL de la forme `dialecte+pilote://utilisateur:motdepasse@hôte:port/base`. Pour éviter les problèmes d'échappement dans le mot de passe, on construit l'URL avec `URL.create()` :

```python
import os
from sqlalchemy import create_engine, text
from sqlalchemy.engine import URL

url = URL.create(
    "mariadb+pymysql",            # dialecte MariaDB, pilote PyMySQL
    username="app_user",
    password=os.environ["DB_PASSWORD"],
    host="localhost",
    port=3306,
    database="boutique",
    query={"charset": "utf8mb4"},
)

engine = create_engine(
    url,
    pool_size=5,          # taille du pool (voir §17.2)
    max_overflow=10,      # connexions supplémentaires temporaires
    pool_pre_ping=True,   # vérifie la connexion avant usage (évite les connexions périmées)
)
```

SQLAlchemy propose un **dialecte `mariadb`** distinct du dialecte `mysql`, qui permet d'activer les comportements propres à MariaDB. Le pilote sous-jacent se choisit dans l'URL : `mariadb+pymysql`, `mariadb+mariadbconnector` (Connector/Python), `mariadb+mysqlconnector`, ou côté MySQL `mysql+mysqldb` (mysqlclient).

### Exécuter du SQL

Quel que soit le pilote choisi, SQLAlchemy expose des **marqueurs nommés `:nom`** via `text()` et se charge de les traduire dans le style attendu par le pilote :

```python
with engine.connect() as conn:
    result = conn.execute(
        text("SELECT id, nom, prix FROM produits WHERE categorie = :cat AND prix < :max"),
        {"cat": categorie, "max": prix_max},
    )
    for row in result.mappings():        # lignes façon dict
        print(f"{row['nom']} : {row['prix']:.2f} €")
```

Cet exemple relève de **SQLAlchemy Core** (langage d'expression SQL). L'usage de l'**ORM** (classes mappées, `Session`, requêtes objet) est présenté au §17.3.2.

> 💡 SQLAlchemy 2.0 propose aussi un mode **asynchrone** (`create_async_engine`), à associer à un pilote async comme `aiomysql` ou `asyncmy` pour les frameworks reposant sur `asyncio`.

## Authentification et MariaDB 12.3

- **`caching_sha2_password`** (§10.5.5) : pris en charge par mysql-connector-python comme par PyMySQL (ce dernier requiert le paquet `cryptography` pour l'échange RSA sur connexion non chiffrée). Une connexion TLS simplifie l'échange.
- **`ed25519`** (§10.5.2) et **PARSEC** (§10.6) : pour ces plugins propres à MariaDB, **MariaDB Connector/Python** (`mariadb`) est le choix naturel, puisqu'il s'appuie sur Connector/C. Les pilotes orientés MySQL ne les reconnaissent pas toujours.

Règle pratique : pour une authentification compatible MySQL, n'importe quel pilote convient ; pour une authentification spécifiquement MariaDB, privilégier le connecteur officiel `mariadb`.

## Tableau comparatif

| Outil | Catégorie | Marqueur | Pur Python | À retenir |
|-------|-----------|----------|------------|-----------|
| **PyMySQL** | pilote DB-API | `%s` | oui | aucune dépendance native, installation triviale |
| **mysql-connector-python** | pilote DB-API | `%s` | oui (ext. C optionnelle) | officiel Oracle, `caching_sha2_password` |

---

## Ce qu'il faut retenir

- Les **pilotes** (PyMySQL, mysql-connector-python, mysqlclient, MariaDB Connector/Python) suivent tous la **DB-API 2.0** ; **SQLAlchemy** est une couche au-dessus, pas un pilote.
- Le marqueur de paramètre est `%s` pour la plupart, **`?` pour MariaDB Connector/Python** : ne jamais l'interpoler à la main (§17.8, §17.9).
- L'**autocommit est désactivé par défaut** : penser à `conn.commit()` pour les écritures.
- Toujours fixer `charset="utf8mb4"`.
- **SQLAlchemy** apporte une URL unifiée, un **pool de connexions intégré** (§17.2) et un dialecte `mariadb` ; son ORM est traité en §17.3.2.
- Pour `ed25519`/PARSEC, préférer le connecteur officiel **`mariadb`** ; pour `caching_sha2_password`, tout pilote moderne convient.

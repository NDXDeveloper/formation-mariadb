# 17.1.4 Node.js : mysql2, mariadb

Dans l'écosystème Node.js, deux connecteurs dominent l'accès à MariaDB, tous deux écrits **entièrement en JavaScript** (famille des implémentations natives au langage, cf. §17.1) :

- **mysql2** — très répandu, compatible avec l'ancien paquet `mysql` tout en étant plus rapide ; il sert de socle à de nombreux outils (Sequelize, par exemple) ;
- **mariadb** — le **connecteur officiel de MariaDB** (*MariaDB Connector/Node.js*), optimisé pour le serveur et ses fonctionnalités propres.

Cette section les présente au niveau de la connexion. L'usage d'un ORM (Sequelize au §17.3.3, Prisma au §17.3.4) est traité séparément.

---

## Le modèle asynchrone

Node.js repose sur des **entrées/sorties non bloquantes** : toute opération réseau, donc toute requête vers la base, est **asynchrone**. Le code moderne s'écrit avec des **Promises** et la syntaxe `async`/`await`. Les deux connecteurs proposent une API basée sur les Promises ; on évite l'ancien style à *callbacks*.

---

## mysql2

### Installation et connexion

```bash
npm install mysql2
```

On importe l'API Promise (`mysql2/promise`) :

```javascript
import mysql from 'mysql2/promise';

const conn = await mysql.createConnection({
  host: 'localhost',
  port: 3306,
  user: 'app_user',
  password: process.env.DB_PASSWORD,   // jamais en dur
  database: 'boutique',
  charset: 'utf8mb4',
});
```

### `query()` ou `execute()` ?

mysql2 propose deux méthodes, et la distinction est importante :

- **`query()`** envoie la requête en protocole texte avec échappement des valeurs côté client ;
- **`execute()`** utilise de **vrais *prepared statements*** (protocole binaire, préparés par le serveur et mis en cache) — c'est la méthode à privilégier.

Les deux utilisent le marqueur positionnel `?` et **retournent un tableau `[lignes, métadonnées]`**, qu'on déstructure :

```javascript
const [rows] = await conn.execute(
  'SELECT id, nom, prix FROM produits WHERE categorie = ? AND prix < ?',
  [categorie, prixMax],
);

for (const row of rows) {
  console.log(`${row.nom} : ${row.prix} €`);
}

await conn.end();
```

### Marqueurs nommés

En activant l'option `namedPlaceholders`, on peut utiliser des marqueurs `:nom` :

```javascript
const conn = await mysql.createConnection({ /* ... */ namedPlaceholders: true });
const [rows] = await conn.execute(
  'SELECT * FROM produits WHERE categorie = :cat',
  { cat: categorie },
);
```

## mariadb — le connecteur officiel

### Installation et connexion

```bash
npm install mariadb
```

L'API est basée sur les Promises par défaut :

```javascript
import * as mariadb from 'mariadb';   // exports nommés : le paquet n'expose pas d'export par défaut

const conn = await mariadb.createConnection({
  host: 'localhost',
  port: 3306,
  user: 'app_user',
  password: process.env.DB_PASSWORD,
  database: 'boutique',
});

const rows = await conn.query(
  'SELECT id, nom, prix FROM produits WHERE categorie = ? AND prix < ?',
  [categorie, prixMax],
);

for (const row of rows) {
  console.log(`${row.nom} : ${row.prix} €`);
}

await conn.end();
```

> ⚠️ **Différence à connaître** : contrairement à mysql2, `query()` du connecteur `mariadb` **retourne directement le tableau de lignes** (et non `[lignes, métadonnées]`). Les métadonnées sont attachées séparément via la propriété `rows.meta`. Du code déplacé d'un connecteur à l'autre trébuche souvent sur ce point.

Comme mysql2, il distingue `query()` (protocole texte) et **`execute()`** (vrais *prepared statements*), et prend en charge l'option `namedPlaceholders`.

### Insertions par lot (batch)

Atout du connecteur officiel : la méthode `batch()` exécute efficacement une insertion répétée sur de nombreuses lignes :

```javascript
await conn.batch(
  'INSERT INTO produits (nom, prix, categorie) VALUES (?, ?, ?)',
  [
    ['Clavier', 49.90, 'peripheriques'],
    ['Souris', 19.90, 'peripheriques'],
  ],
);
```

L'identifiant généré par un `INSERT` se lit dans l'objet résultat :

```javascript
const result = await conn.query(
  'INSERT INTO produits (nom, prix, categorie) VALUES (?, ?, ?)',
  ['Clavier', 49.90, 'peripheriques'],
);
const nouvelId = result.insertId;   // voir l'avertissement sur les grands entiers ci-dessous
```

## Pooling

En production, on ne crée pas une connexion par requête : on utilise un **pool**, fourni nativement par les deux connecteurs.

**Avec mysql2 :**

```javascript
const pool = mysql.createPool({
  host: 'localhost',
  user: 'app_user',
  password: process.env.DB_PASSWORD,
  database: 'boutique',
  waitForConnections: true,
  connectionLimit: 10,
});

const [rows] = await pool.execute('SELECT id, nom FROM produits WHERE id = ?', [id]);
```

**Avec le connecteur mariadb :**

```javascript
const pool = mariadb.createPool({
  host: 'localhost',
  user: 'app_user',
  password: process.env.DB_PASSWORD,
  database: 'boutique',
  connectionLimit: 10,
});

let conn;
try {
  conn = await pool.getConnection();
  const rows = await conn.query('SELECT id, nom FROM produits WHERE id = ?', [id]);
} finally {
  if (conn) conn.release();   // rend la connexion au pool
}
```

On peut aussi appeler directement `pool.query(...)`. Le dimensionnement et la gestion du pool sont détaillés au §17.2.

## Transactions

```javascript
const conn = await pool.getConnection();
try {
  await conn.beginTransaction();
  await conn.query('UPDATE comptes SET solde = solde - ? WHERE id = ?', [montant, source]);
  await conn.query('UPDATE comptes SET solde = solde + ? WHERE id = ?', [montant, destination]);
  await conn.commit();
} catch (err) {
  await conn.rollback();
  throw err;
} finally {
  conn.release();
}
```

## Piège : les grands entiers (BIGINT)

Particularité propre à JavaScript : le type `Number` est un flottant 64 bits, exact uniquement jusqu'à 2⁵³−1 (`Number.MAX_SAFE_INTEGER`). Or une colonne `BIGINT` de MariaDB peut dépasser cette limite, ce qui entraînerait une **perte de précision** silencieuse.

Les deux connecteurs offrent des options pour traiter ce cas :

- le connecteur **mariadb** peut renvoyer les `BIGINT` sous forme de `BigInt` JavaScript afin de préserver la précision (options telles que `bigIntAsNumber`, `insertIdAsNumber`, `decimalAsNumber` pour ajuster ce comportement) ;
- **mysql2** renvoie par défaut un `Number` ; on configure `supportBigNumbers` et `bigNumberStrings` pour obtenir les grands nombres sous forme de chaîne.

Les valeurs par défaut varient selon la version : pour toute colonne `BIGINT` susceptible de grands entiers (identifiants notamment), vérifier le comportement et activer l'option adaptée.

## Authentification et MariaDB 12.3

- **`caching_sha2_password`** (§10.5.5) : pris en charge par mysql2 comme par le connecteur mariadb, ce qui facilite la connexion à un MariaDB 12.3 configuré avec ce plugin hérité de MySQL.
- **`ed25519`** (§10.5.2) et **PARSEC** (§10.6) : pour ces plugins propres à MariaDB, le **connecteur officiel `mariadb`** est le choix naturel ; mysql2, orienté MySQL, ne les reconnaît pas nativement.

Le chiffrement TLS se configure via l'option `ssl` (objet de configuration ou `{ rejectUnauthorized: true }` avec un certificat de CA), voir §10.7.

---

## mysql2 ou mariadb ?

- **mysql2** : écosystème très large, compatible avec l'ancien paquet `mysql`, employé par des ORM comme Sequelize. À privilégier pour une compatibilité maximale ou un code existant orienté MySQL.
- **mariadb** : connecteur officiel, optimisé pour le serveur, avec les fonctionnalités spécifiques (authentification `ed25519`, insertions `batch`, gestion fine des types). À privilégier dès lors que la cible est MariaDB.

---

## Ce qu'il faut retenir

- Node.js est **asynchrone** : utiliser les Promises et `async`/`await` avec mysql2 ou le connecteur `mariadb` (tous deux purs JavaScript).
- Préférer **`execute()`** (vrais *prepared statements* mis en cache) à `query()`, avec le marqueur `?` (ou `:nom` via `namedPlaceholders`).
- **Attention à la valeur de retour** : mysql2 renvoie `[lignes, métadonnées]`, le connecteur `mariadb` renvoie directement les lignes.
- En production, passer par un **pool** (§17.2) et libérer les connexions (`release()`).
- Gérer le **piège des `BIGINT`** au-delà de 2⁵³−1 via les options de typage des grands nombres.
- Pour `ed25519`/PARSEC, préférer le connecteur officiel **`mariadb`** ; `caching_sha2_password` fonctionne avec les deux.

# 4.7.2 Fonctions JSON (`JSON_EXTRACT`, `JSON_SET`, `JSON_ARRAY`, etc.)

> **Chapitre 4 — Concepts Avancés SQL** · Niveau : Avancé  
> Fonctions JSON depuis 10.2.x · `JSON_TABLE` depuis 10.6 · Référence : **MariaDB 12.3 LTS**

MariaDB fournit une bibliothèque complète de fonctions pour manipuler le JSON. Plutôt que de toutes les énumérer, cette section les présente **par usage** : extraire, construire, modifier, inspecter, mettre en forme, et transformer un document en table. Un principe traverse l'ensemble : ces fonctions **renvoient un nouveau document** ou une valeur ; elles ne modifient jamais la colonne en place — pour persister un changement, il faut un `UPDATE`.

## 🎯 Objectif de la section

Connaître les principales fonctions JSON regroupées par finalité, comprendre les distinctions essentielles (`JSON_VALUE` vs `JSON_QUERY`, `JSON_SET` vs `JSON_INSERT` vs `JSON_REPLACE`), et savoir transformer du JSON en lignes avec `JSON_TABLE`.

## Extraire des données

Trois fonctions extraient une donnée à partir d'un **chemin** (les expressions de chemin sont détaillées au § 4.8).

**`JSON_EXTRACT(doc, chemin[, chemin…])`** est l'extracteur général. Il renvoie la valeur correspondant au chemin — ou un tableau si plusieurs chemins sont fournis —, et `NULL` si rien ne correspond ou si un argument est `NULL`. Particularité : les **chaînes sont restituées entre guillemets**.

```sql
SELECT JSON_EXTRACT('{"nom":"Monty","age":30}', '$.nom');
-- "Monty"   (avec les guillemets)
```

Pour retirer ces guillemets, on applique `JSON_UNQUOTE` — ce que fait justement l'opérateur raccourci `->>` (§ 4.7.3).

Les deux extracteurs conformes au standard SQL se distinguent par le **type de ce qu'ils renvoient** :

- **`JSON_VALUE(doc, chemin)`** ne renvoie qu'une **valeur scalaire** (et `NULL` si la cible est un objet ou un tableau) ;
- **`JSON_QUERY(doc, chemin)`** ne renvoie qu'un **objet ou un tableau** (et `NULL` si la cible est un scalaire).

```sql
SET @j = '{"x":[0,1], "y":"texte", "z":42}';
SELECT JSON_VALUE(@j, '$.z');   -- 42      (scalaire)
SELECT JSON_VALUE(@j, '$.x');   -- NULL    (x est un tableau)
SELECT JSON_QUERY(@j, '$.x');   -- [0, 1]  (objet/tableau)
SELECT JSON_QUERY(@j, '$.z');   -- NULL    (z est un scalaire)
```

`JSON_VALUE` est particulièrement commode pour **filtrer** sur un champ JSON dans un `WHERE` :

```sql
SELECT id, nom
FROM lieux
WHERE JSON_VALUE(attributs, '$.details.annee') = 1924;
```

> 💡 Un tel filtre **analyse le document à chaque ligne**. Pour de gros volumes, on extrait le champ dans une colonne virtuelle indexée (§ 4.10).

## Construire des documents

- **`JSON_OBJECT(clé, valeur, …)`** construit un objet, **`JSON_ARRAY(val, …)`** un tableau :

```sql
SELECT JSON_OBJECT('nom', 'Monty', 'actif', TRUE);  -- {"nom": "Monty", "actif": true}
SELECT JSON_ARRAY(1, 'deux', 3);                     -- [1, "deux", 3]
```

- **`JSON_QUOTE(texte)`** transforme une chaîne en valeur JSON entre guillemets (échappement compris).
- Côté **agrégation**, `JSON_ARRAYAGG(expr)` rassemble les valeurs d'un groupe en un tableau JSON, et `JSON_OBJECTAGG(clé, valeur)` en un objet — l'équivalent JSON de `GROUP_CONCAT`.

```sql
SELECT categorie, JSON_ARRAYAGG(nom) AS produits
FROM produits GROUP BY categorie;
```

## Modifier un document

Trois fonctions modifient un document, avec des sémantiques distinctes :

- **`JSON_SET(doc, chemin, val, …)`** : insère **ou** met à jour (comportement « upsert ») ;
- **`JSON_INSERT(doc, chemin, val, …)`** : insère **uniquement** si la clé n'existe pas (ne remplace pas) ;
- **`JSON_REPLACE(doc, chemin, val, …)`** : met à jour **uniquement** une clé existante (n'ajoute rien).

S'y ajoutent **`JSON_REMOVE`** (supprimer un élément), **`JSON_ARRAY_APPEND`** / **`JSON_ARRAY_INSERT`** (manipuler un tableau), et la fusion de documents : **`JSON_MERGE_PATCH`** (fusion façon RFC 7396, la dernière valeur l'emporte, un `null` supprime) et **`JSON_MERGE_PRESERVE`** (combine en conservant les deux côtés). `JSON_MERGE` est un alias déprécié.

> ⚠️ **Ces fonctions ne modifient pas la colonne.** Elles renvoient un nouveau document. Pour enregistrer la modification, il faut l'affecter via un `UPDATE` :

```sql
SELECT JSON_SET('{"a":1}', '$.b', 2);          -- {"a": 1, "b": 2}  (résultat seulement)

UPDATE produits
SET attributs = JSON_SET(attributs, '$.stock', 0)
WHERE id = 1;                                   -- persiste le changement
```

## Inspecter et interroger

Ces fonctions renseignent sur le contenu ou la structure :

- **`JSON_VALID(doc)`** : document valide (1/0) — c'est la fonction de la contrainte automatique (§ 4.7.1) ;
- **`JSON_TYPE(val)`** : type d'une valeur (`OBJECT`, `ARRAY`, `STRING`, `INTEGER`, `BOOLEAN`, `NULL`…) ;
- **`JSON_LENGTH(doc[, chemin])`** : nombre d'éléments ; **`JSON_DEPTH(doc)`** : profondeur maximale ;
- **`JSON_KEYS(doc[, chemin])`** : tableau des clés d'un objet ;
- **`JSON_CONTAINS(doc, val[, chemin])`** : le document contient-il une valeur ? **`JSON_CONTAINS_PATH(doc, mode, chemin…)`** : un chemin existe-t-il ? **`JSON_EXISTS(doc, chemin)`** : un chemin est-il présent ? ;
- **`JSON_SEARCH(doc, mode, valeur)`** : renvoie le chemin où figure une valeur ;
- **`JSON_OVERLAPS(doc1, doc2)`** : deux documents ont-ils des éléments en commun ?

```sql
SELECT JSON_TYPE('[1,2,3]');                          -- ARRAY
SELECT JSON_LENGTH('{"a":1,"b":2}');                  -- 2
SELECT JSON_CONTAINS('{"r":["a","b"]}', '"a"', '$.r'); -- 1 (présent)
```

## Mettre en forme et comparer

- **`JSON_DETAILED(doc)`** ré-indente un document pour le rendre lisible (`JSON_PRETTY` en est l'alias de compatibilité MySQL) ; **`JSON_COMPACT(doc)`** fait l'inverse en supprimant les espaces.
- **`JSON_NORMALIZE(doc)`** produit une **forme canonique** : c'est l'outil pour comparer **sémantiquement** deux documents, là où l'égalité par défaut est purement textuelle (§ 4.7.1). Deux documents équivalents mais formatés différemment se normalisent en une même chaîne.

```sql
SELECT JSON_NORMALIZE('{ "a":1, "b":2 }') = JSON_NORMALIZE('{"b":2,"a":1}');  -- 1
```

- **`JSON_UNQUOTE(val)`** retire les guillemets d'une valeur scalaire extraite.

## Transformer en table : `JSON_TABLE`

`JSON_TABLE` (depuis MariaDB 10.6) est la fonction la plus puissante : elle **convertit un document JSON en table relationnelle**, ce qui permet de joindre, filtrer et agréger des données JSON comme des lignes ordinaires. Elle s'emploie là où une table peut figurer — dans le `FROM`, ainsi que dans les `UPDATE`/`DELETE` multi-tables.

Soit une table `personnes` dont la colonne `animaux` contient un tableau d'objets :

```sql
-- animaux = [{"type":"chien","nom":"Duke"}, {"type":"chat","nom":"Fluffy"}]
SELECT p.id, t.type_animal, t.nom_animal
FROM personnes AS p,
     JSON_TABLE(
         p.animaux,
         '$[*]'                          -- une ligne par élément du tableau
         COLUMNS (
             type_animal VARCHAR(20) PATH '$.type',
             nom_animal  VARCHAR(50) PATH '$.nom'
         )
     ) AS t;
```

Chaque élément du tableau JSON devient une **ligne**, chaque colonne étant extraite par son chemin. La clause **`NESTED PATH`** gère les tableaux imbriqués (un élément contenant lui-même un tableau), **`FOR ORDINALITY`** ajoute un numéro de ligne, et les clauses `ON EMPTY` / `ON ERROR` définissent le comportement en cas d'absence ou d'erreur.

## Tableau récapitulatif

| Usage              | Fonctions principales                                                                 |
|--------------------|---------------------------------------------------------------------------------------|
| **Extraction**     | `JSON_EXTRACT`, `JSON_VALUE` (scalaire), `JSON_QUERY` (objet/tableau), `JSON_UNQUOTE` |
| **Construction**   | `JSON_OBJECT`, `JSON_ARRAY`, `JSON_QUOTE`, `JSON_ARRAYAGG`, `JSON_OBJECTAGG`           |
| **Modification**   | `JSON_SET`, `JSON_INSERT`, `JSON_REPLACE`, `JSON_REMOVE`, `JSON_MERGE_PATCH`, `JSON_MERGE_PRESERVE` |
| **Inspection**     | `JSON_VALID`, `JSON_TYPE`, `JSON_LENGTH`, `JSON_DEPTH`, `JSON_KEYS`, `JSON_CONTAINS`, `JSON_EXISTS`, `JSON_SEARCH`, `JSON_OVERLAPS` |
| **Mise en forme**  | `JSON_DETAILED`, `JSON_COMPACT`, `JSON_NORMALIZE`                                      |
| **Tabularisation** | `JSON_TABLE`                                                                          |

## Points clés à retenir

Les fonctions JSON se rangent par usage : **extraire** (`JSON_EXTRACT`, et surtout `JSON_VALUE` pour un scalaire vs `JSON_QUERY` pour un objet/tableau), **construire** (`JSON_OBJECT`, `JSON_ARRAY`, et les agrégats `JSON_ARRAYAGG`/`JSON_OBJECTAGG`), **modifier** (`JSON_SET` en upsert, `JSON_INSERT` en ajout seul, `JSON_REPLACE` en mise à jour seule, plus les fusions), **inspecter** (`JSON_VALID`, `JSON_TYPE`, `JSON_CONTAINS`…) et **mettre en forme** (`JSON_DETAILED`, `JSON_NORMALIZE` pour la comparaison sémantique). Les fonctions de modification **renvoient un nouveau document** : il faut un `UPDATE` pour persister. Enfin, **`JSON_TABLE`** convertit un document en table relationnelle, ouvrant le JSON aux jointures et agrégations classiques.

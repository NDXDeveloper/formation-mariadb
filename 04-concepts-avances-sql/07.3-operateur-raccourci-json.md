# 4.7.3 Opérateur raccourci (`->>`)

> **Chapitre 4 — Concepts Avancés SQL** · Niveau : Avancé  
> Référence : **MariaDB 12.3 LTS**

Dans MySQL — comme dans PostgreSQL ou SQLite — les opérateurs `->` et `->>` offrent une syntaxe abrégée pour extraire une valeur d'un document JSON, plus concise que les fonctions. Cette section explique ce qu'ils font, **puis précise un point essentiel** : MariaDB **ne les implémente pas**. Nous présentons donc les équivalents fonctionnels à employer en MariaDB, et l'incidence sur la migration.

## 🎯 Objectif de la section

Comprendre le rôle des opérateurs `->` et `->>`, savoir qu'ils sont absents de MariaDB, et maîtriser les fonctions équivalentes (`JSON_VALUE`, `JSON_EXTRACT`, `JSON_UNQUOTE`).

## Les opérateurs `->` et `->>` (origine MySQL)

Dans les SGBD qui les proposent, ces opérateurs s'emploient sous la forme `colonne -> '$.chemin'`, où l'opérande gauche est le document et l'opérande droit une expression de chemin :

- **`->`** est un raccourci de `JSON_EXTRACT()` : il renvoie la valeur **brute**, les chaînes restant **entre guillemets** ;
- **`->>`** va plus loin en **déguillemetant** le résultat — l'équivalent de `JSON_UNQUOTE(JSON_EXTRACT())`.

La différence se voit surtout sur les chaînes :

| Valeur JSON au chemin | `->` (brut) | `->>` (déguillemeté) |
|-----------------------|-------------|----------------------|
| `123`                 | `123`       | `123`                |
| `"texte"`             | `"texte"`   | `texte`              |
| `[6, 7, 8]`           | `[6, 7, 8]` | `[6, 7, 8]`          |

## ⚠️ MariaDB n'implémente pas ces opérateurs

Point déterminant : **MariaDB ne prend pas en charge les opérateurs `->` et `->>`** (demande MDEV-13594, ouverte depuis 2017 et non implémentée, y compris dans la série 12.x). Écrire une requête de ce type :

```sql
SELECT donnees ->> '$.nom' FROM produits;   -- ❌ erreur de syntaxe en MariaDB
```

provoque une **erreur de syntaxe**. Il s'agit d'un écart connu de compatibilité avec MySQL : la documentation de MariaDB le recense explicitement parmi les différences entre les deux SGBD.

## Les équivalents en MariaDB

MariaDB couvre les mêmes besoins par des **fonctions** (étudiées au § 4.7.2). La correspondance est directe :

| Opérateur (MySQL/PostgreSQL) | Effet                                   | Équivalent MariaDB                                   |
|------------------------------|-----------------------------------------|------------------------------------------------------|
| `doc -> '$.x'`               | valeur JSON brute (chaînes entre guillemets) | `JSON_EXTRACT(doc, '$.x')` (ou `JSON_QUERY` pour un objet/tableau) |
| `doc ->> '$.x'`              | valeur scalaire déguillemetée           | `JSON_VALUE(doc, '$.x')` (ou `JSON_UNQUOTE(JSON_EXTRACT(doc, '$.x'))`) |

En pratique, c'est **`JSON_VALUE`** qui joue le rôle de `->>` pour le cas le plus courant — extraire un scalaire lisible, par exemple pour l'afficher ou le filtrer :

```sql
-- MySQL   : SELECT donnees ->> '$.nom' AS nom FROM produits;
-- MariaDB :
SELECT JSON_VALUE(donnees, '$.nom') AS nom FROM produits;
```

Et **`JSON_EXTRACT`** (ou `JSON_QUERY`) tient le rôle de `->`, en conservant la forme JSON :

```sql
-- MySQL   : SELECT donnees -> '$.tags' AS tags FROM produits;
-- MariaDB :
SELECT JSON_EXTRACT(donnees, '$.tags') AS tags FROM produits;
```

> 💡 **Nuance.** `JSON_VALUE` ne renvoie qu'un **scalaire** (et `NULL` sur un objet ou un tableau), alors que `->>` de MySQL, appliqué à un objet ou un tableau, en renvoie le texte. Pour reproduire exactement `->>` dans tous les cas, on utilise l'expansion littérale `JSON_UNQUOTE(JSON_EXTRACT(doc, '$.x'))`. Pour le besoin habituel — un scalaire déguillemeté —, `JSON_VALUE` suffit et reste le plus lisible.

## Incidence sur la migration

Cet écart a une conséquence concrète : tout code provenant de **MySQL, PostgreSQL ou SQLite** qui utilise `->` ou `->>` doit être **réécrit** avec les fonctions équivalentes lors d'un portage vers MariaDB. Le point mérite une attention particulière car certains **ORM** ou générateurs de requêtes traduisent un accès du type `donnees['x']` en `->>` (ou en `JSON_EXTRACT`), avec des comportements qui diffèrent d'un SGBD à l'autre. Ces aspects sont approfondis au chapitre 19 (migration).

## Points clés à retenir

Les opérateurs `->` (raccourci de `JSON_EXTRACT`) et `->>` (raccourci de `JSON_UNQUOTE(JSON_EXTRACT)`) appartiennent à MySQL (et PostgreSQL, SQLite) : **MariaDB ne les implémente pas** (MDEV-13594), et leur emploi y produit une erreur de syntaxe. On les remplace par des fonctions : **`JSON_VALUE`** pour obtenir un scalaire déguillemeté (équivalent de `->>`), **`JSON_EXTRACT`** ou **`JSON_QUERY`** pour la valeur JSON brute (équivalent de `->`), et l'expansion `JSON_UNQUOTE(JSON_EXTRACT(...))` pour une équivalence exacte. Ce point de compatibilité impose de réécrire le code JSON abrégé lors d'une migration vers MariaDB (chapitre 19).

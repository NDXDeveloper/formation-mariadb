# 3.7.1 Fonctions de compatibilité Oracle (TO_DATE, TRUNC, TO_NUMBER, TO_CHAR avec format FM) 🆕

> **Chapitre 3 : Requêtes SQL Intermédiaires** · Section 3.7.1  
> 🆕 Compléments apportés par la série **MariaDB 12.x** · Version de référence : **MariaDB 12.3 LTS**

---

## Contexte

Oracle dispose d'une famille de fonctions de **conversion** — `TO_CHAR`, `TO_DATE`, `TO_NUMBER`, `TRUNC` — omniprésentes dans son SQL. Pour faciliter les migrations, MariaDB en propose des équivalents. Plusieurs de ces capacités sont **apparues ou ont été complétées dans la série 12.x**, ce qui justifie le marquage 🆕 de cette section :

- le modificateur de format **`FM` de `TO_CHAR` a été ajouté en MariaDB 12.0** ; MariaDB prend en charge le spécificateur FM pour supprimer le remplissage depuis la version 12.0.
- **`TO_NUMBER` est disponible depuis MariaDB 12.2** ; cette fonction compatible Oracle analyse une chaîne selon un format donné et renvoie une valeur numérique.
- **`TRUNC` sur une date a été introduite en MariaDB 12.2** (en mode Oracle).
- **`TO_DATE` elle-même est une nouveauté de MariaDB 12.3** (disponible aussi hors mode Oracle).

Seule la fonction `TO_CHAR` est plus ancienne — apparue avec la **10.6** ; c'est son modificateur `FM` qui relève de la série 12.x. Ces fonctions sont des **outils de compatibilité** : en SQL MariaDB natif, on emploie les fonctions de la [section 3.7](07-fonctions-dates-heures.md) (`DATE_FORMAT`, `STR_TO_DATE`) et de la [section 3.6](06-fonctions-chaines.md) (`FORMAT`).

---

## Le modèle de format Oracle

Toutes ces fonctions partagent le **modèle de format d'Oracle**, très différent des spécificateurs `%` natifs vus en [section 3.7](07-fonctions-dates-heures.md). Là où `DATE_FORMAT` écrit `'%d/%m/%Y'`, le modèle Oracle écrit `'DD/MM/YYYY'` :

| Élément | Signification | Élément | Signification |
|---------|---------------|---------|---------------|
| `YYYY` | année sur 4 chiffres | `MI` | minutes |
| `YY` / `RR` | année sur 2 chiffres | `SS` | secondes |
| `MM` | mois `01`–`12` | `HH24` | heure `00`–`23` |
| `MON` / `MONTH` | mois abrégé / complet | `HH` / `HH12` | heure `01`–`12` |
| `DD` | jour du mois | `DY` / `DAY` | jour abrégé / complet |

C'est ce vocabulaire que `TO_CHAR` et `TO_DATE` interprètent.

---

## TO_CHAR : date → chaîne

`TO_CHAR(expr[, format])` convertit une expression de type date, datetime, time ou timestamp en chaîne de caractères. Le format par défaut est `'YYYY-MM-DD HH24:MI:SS'`.

```sql
SELECT TO_CHAR('2026-06-03', 'DD/MM/YYYY')          AS date_fr;   -- '03/06/2026'
SELECT TO_CHAR('2026-06-03 14:30:00', 'HH24:MI:SS') AS heure;     -- '14:30:00'
```

Deux particularités MariaDB à connaître :

- **Capitalisation fixe.** Là où Oracle module la casse selon celle du format (`'DAY'` → `TUESDAY`, `'Day'` → `Tuesday`), MariaDB utilise toujours une capitalisation à l'initiale, quelle que soit la casse du format : `'DAY'`, `'Day'` et `'day'` produisent tous `Tuesday`.
- **Pas de conversion de nombres.** Sous Oracle, TO_CHAR peut aussi convertir des nombres en chaînes, mais ce n'est pas pris en charge par MariaDB et provoque une erreur. Pour formater un nombre, utilisez `FORMAT` ou `LPAD` ([section 3.6](06-fonctions-chaines.md)).

---

## Le modificateur de format FM 🆕

C'est l'apport mis en avant dans cette section. Par défaut, les noms de jour et de mois renvoyés par `TO_CHAR` sont **complétés par des espaces** jusqu'à la largeur du nom le plus long de la locale `lc_time_names`. Le modificateur **`FM`** (« Fill Mode ») supprime ce remplissage. `FM` bascule le remplissage de tous les composants qui le suivent : un premier `FM` le désactive, un deuxième le réactive, et ainsi de suite — un nombre impair de `FM` désactive le remplissage.

```sql
-- lc_time_names = en_US ; les noms de jour sont remplis jusqu'à 9 caractères (« Wednesday »)
SELECT CONCAT('[', TO_CHAR('2026-01-06', 'DAY'),   ']') AS sans_fm;  -- '[Tuesday  ]'
SELECT CONCAT('[', TO_CHAR('2026-01-06', 'FMDAY'), ']') AS avec_fm;  -- '[Tuesday]'
```

Sans `FM`, `Tuesday` (7 caractères) est complété par deux espaces pour atteindre la largeur de `Wednesday`. Avec `FMDAY`, le remplissage disparaît. Ce comportement, ajouté en **12.0**, aligne `TO_CHAR` sur la sémantique Oracle du `fm`.

---

## TO_DATE : chaîne → date

`TO_DATE(chaîne, format)` réalise l'opération inverse de `TO_CHAR`. Ajoutée en **MariaDB 12.3** pour la prise en charge d'Oracle, elle accepte les mêmes éléments de format que `TO_CHAR` (à quelques exceptions près) et renvoie une valeur datetime.

```sql
SELECT TO_DATE('03/06/2026', 'DD/MM/YYYY') AS d;   -- 2026-06-03 00:00:00
```

C'est l'équivalent Oracle de `STR_TO_DATE` ([section 3.7](07-fonctions-dates-heures.md)), avec le modèle de format Oracle au lieu des spécificateurs `%`.

---

## TO_NUMBER : chaîne → nombre 🆕

`TO_NUMBER(chaîne[, format])` analyse une chaîne pour en extraire un nombre. Disponible depuis MariaDB 12.2, elle convertit une expression vers le type numérique et renvoie le type DOUBLE pour toutes les signatures.

```sql
SELECT TO_NUMBER('1234.56') AS n;   -- 1234.56
```

La fonction accepte un format Oracle optionnel pour analyser des chaînes formatées (séparateurs de milliers, signes, etc.). Cependant, à la date de cette version, une partie des composants de format reste en cours d'implémentation côté MariaDB (suivie par le ticket MDEV-36978) : vérifiez le comportement des formats avancés sur votre version précise. Sans format, `TO_NUMBER` reconnaît la notation décimale standard.

---

## TRUNC : tronquer une date 🆕

Attention à ne pas confondre deux fonctions de noms proches :

- **`TRUNCATE(nombre, décimales)`** — fonction **native** de longue date qui tronque un **nombre** (ex. `TRUNCATE(85.253, 1)` → `85.2`).
- **`TRUNC(date[, unité])`** — fonction de **compatibilité Oracle** qui tronque une **date**.

Pour `TRUNC` sur une date : elle est disponible depuis la version 12.2, uniquement en mode de compatibilité Oracle ; dans les versions antérieures, on utilisait CAST, DATE et DATE_FORMAT. En mode Oracle, elle tronque une valeur de date à l'unité de mesure indiquée et renvoie un DATETIME. Les unités acceptées tronquent au jour (DD, DDD, J), au mois (MM, MON, MONTH, RM) ou à l'année (YEAR, YYYY, YY…) ; sans unité, la troncature se fait au jour le plus proche.

```sql
-- Mode Oracle requis
SELECT TRUNC('2026-06-03 14:30:00', 'MM')   AS debut_mois;   -- 2026-06-01 00:00:00
SELECT TRUNC('2026-06-03 14:30:00', 'YYYY') AS debut_annee;  -- 2026-01-01 00:00:00
SELECT TRUNC('2026-06-03 14:30:00')         AS jour;         -- 2026-06-03 00:00:00
```

`TRUNC(date, 'MM')` ramène au premier jour du mois — une opération très courante en reporting, qu'on exprimerait nativement par `DATE_FORMAT(d, '%Y-%m-01')`.

## Tableau récapitulatif

| Fonction Oracle | Rôle | Équivalent natif MariaDB | Disponibilité |
|-----------------|------|--------------------------|---------------|
| `TO_CHAR(date, fmt)` | date → chaîne | `DATE_FORMAT` | 10.6 ; **`FM` en 12.0** 🆕 |
| `TO_DATE(str, fmt)` | chaîne → date | `STR_TO_DATE` | **12.3** 🆕 |
| `TO_NUMBER(str[, fmt])` | chaîne → nombre | `CAST(... AS DECIMAL)` | **12.2** 🆕 |
| `TRUNC(date[, unité])` | tronquer une date | `DATE_FORMAT` / `CAST(DATE(...))` | **12.2, mode Oracle** 🆕 |
| `TRUNCATE(nb, déc.)` | tronquer un nombre | *(natif)* | de longue date |

## Recommandation

Ces fonctions répondent à un objectif précis : **exécuter et migrer du code Oracle** sans tout réécrire. Leurs limites (capitalisation fixe de `TO_CHAR`, absence de conversion de nombres par `TO_CHAR`, `TRUNC` restreinte au mode Oracle, composants de format `TO_NUMBER` encore partiels) confirment qu'elles ne remplacent pas les fonctions natives pour du **code MariaDB neuf**. La règle reste celle de la [section 3.3.5](03.5-oracle-outer-join.md) sur l'opérateur `( + )` : **les connaître pour la migration, écrire le nouveau code avec les fonctions natives** (`DATE_FORMAT`, `STR_TO_DATE`, `FORMAT`, `TRUNCATE`). La migration depuis Oracle est traitée globalement en [section 19.2.1](../19-migration-compatibilite/02.1-depuis-oracle.md).

---

## À retenir

- `TO_CHAR`/`TO_DATE`/`TO_NUMBER`/`TRUNC` sont des fonctions de **compatibilité Oracle**, utilisant le **modèle de format Oracle** (`'DD/MM/YYYY'`) et non les spécificateurs `%` natifs.
- Nouveautés 12.x : **`TO_DATE`** (12.3), le modificateur **`FM`** de `TO_CHAR` (12.0, supprime le remplissage), **`TO_NUMBER`** (12.2, renvoie un `DOUBLE`), et **`TRUNC` sur date** (12.2, **mode Oracle uniquement**).
- `TO_CHAR` MariaDB capitalise toujours à l'initiale et **ne convertit pas les nombres** (utiliser `FORMAT`).
- Ne pas confondre **`TRUNC`** (date, Oracle) et **`TRUNCATE`** (nombre, natif).
- Équivalents natifs à privilégier en code neuf : `DATE_FORMAT`, `STR_TO_DATE` ([3.7](07-fonctions-dates-heures.md)), `FORMAT` ([3.6](06-fonctions-chaines.md)).

# 8.3.2 Triggers `INSERT`, `UPDATE`, `DELETE`

La deuxième dimension d'un trigger est l'**événement** qui le déclenche : `INSERT`, `UPDATE` ou `DELETE`. Combiné aux deux timings vus en 8.3.1, cela donne six déclencheurs possibles (`BEFORE INSERT`, `AFTER UPDATE`, etc.). Au-delà du moment d'exécution, le choix de l'événement a une conséquence directe : il détermine quelles données la ligne expose au trigger, à travers les pseudo-enregistrements `OLD` et `NEW`.

## Les valeurs disponibles selon l'événement

La logique est intuitive : `OLD` représente la ligne *avant* l'opération, `NEW` la ligne *après*. Selon l'événement, l'une, l'autre, ou les deux ont un sens.

| Événement | `OLD` | `NEW` |
|---|---|---|
| `INSERT` | indisponible | la ligne insérée |
| `UPDATE` | la ligne avant | la ligne après |
| `DELETE` | la ligne supprimée | indisponible |

À l'`INSERT`, aucune ligne ne préexiste : seul `NEW` (la nouvelle ligne) est accessible. Au `DELETE`, aucune ligne ne subsiste : seul `OLD` (la ligne qui disparaît) est accessible. L'`UPDATE` est le seul cas où les deux coexistent, ce qui permet de comparer l'avant et l'après. Pour rappel (voir 8.3.1), `NEW` n'est *modifiable* qu'en `BEFORE` ; `OLD` est toujours en lecture seule. Le détail de ces pseudo-enregistrements est l'objet de la section 8.3.3.

## Un trigger par événement

L'exemple suivant illustre les trois événements sur une même table `produits`, en alimentant une table d'historique `produits_audit`. Chaque trigger n'utilise que les valeurs disponibles pour son événement.

```sql
DELIMITER $$

-- INSERT : seul NEW est disponible
CREATE OR REPLACE TRIGGER trg_produits_after_insert
    AFTER INSERT ON produits
    FOR EACH ROW
BEGIN
    INSERT INTO produits_audit (produit_id, action, prix_avant, prix_apres)
    VALUES (NEW.id, 'INSERT', NULL, NEW.prix);
END $$

-- UPDATE : OLD et NEW sont disponibles
CREATE OR REPLACE TRIGGER trg_produits_after_update
    AFTER UPDATE ON produits
    FOR EACH ROW
BEGIN
    INSERT INTO produits_audit (produit_id, action, prix_avant, prix_apres)
    VALUES (NEW.id, 'UPDATE', OLD.prix, NEW.prix);
END $$

-- DELETE : seul OLD est disponible
CREATE OR REPLACE TRIGGER trg_produits_after_delete
    AFTER DELETE ON produits
    FOR EACH ROW
BEGIN
    INSERT INTO produits_audit (produit_id, action, prix_avant, prix_apres)
    VALUES (OLD.id, 'DELETE', OLD.prix, NULL);
END $$

DELIMITER ;
```

## Empêcher ou archiver avant une suppression

Le `BEFORE DELETE` mérite une mention particulière : c'est le dernier moment où la ligne existe encore. On peut s'en servir pour l'**archiver** avant qu'elle ne disparaisse, ou pour **bloquer la suppression** sous condition, en levant une erreur avec `SIGNAL` (détaillé en 8.6).

```sql
DELIMITER $$

CREATE OR REPLACE TRIGGER trg_produits_before_delete
    BEFORE DELETE ON produits
    FOR EACH ROW
BEGIN
    IF OLD.protege = 1 THEN
        SIGNAL SQLSTATE '45000'
            SET MESSAGE_TEXT = 'Produit protégé : suppression interdite';
    END IF;
END $$

DELIMITER ;
```

## Quelles instructions déclenchent les triggers ?

Tous les chemins menant à une modification de ligne ne déclenchent pas les triggers de la même façon ; quelques cas méritent l'attention :

- `LOAD DATA` déclenche les triggers d'`INSERT` pour chaque ligne chargée.
- `REPLACE` se comporte, lorsqu'une ligne existante est remplacée, comme un `DELETE` suivi d'un `INSERT` : il active donc les triggers `DELETE` puis `INSERT`, et **non** ceux d'`UPDATE`.
- `INSERT … ON DUPLICATE KEY UPDATE` déclenche les triggers d'`INSERT` quand une ligne est insérée, et ceux d'`UPDATE` quand une ligne existante est mise à jour à la place.
- `TRUNCATE TABLE` **ne déclenche aucun trigger** : ce n'est pas une suppression ligne à ligne, mais une opération qui vide la table en la contournant. C'est un piège fréquent pour l'audit.
- Les **actions en cascade des clés étrangères** (`ON DELETE CASCADE`, `ON UPDATE CASCADE`) **n'activent pas** les triggers sur les tables enfants modifiées — autre point de vigilance lorsqu'on s'appuie sur des triggers pour tracer les changements.
